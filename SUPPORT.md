# Support

Use GitHub issues for reproducible bugs, compatibility reports, and provider
questions. Include the host-agent version, operating system/architecture, MCP
client version, and the failing tool or startup stage. Remove tokens,
passwords, hostnames, public URLs, and full command output that may contain
secrets.

Use the private security-reporting path described in `SECURITY.md` for
vulnerabilities. Security reports must not be filed in public issues.
